## Порядок работы

1. Код, примеры, `tests/rules/*.test.cjs`
2. `npm test`
3. Правила по типу изменения (см. матрицу): `markdownlint-project` → `ts-*` / `js-*` при необходимости — в обоих каталогах (`.cursor/rules/*.mdc` и `.claude/rules/*.md`)
4. `AGENTS.md` → `README.md`
5. Финальный чеклист

Workflow — канон в [AGENTS.md](../../AGENTS.md); правила не дублируют шаги build/test/sync.

## Антипаттерны

- Обновить docs без `npm test` после изменения логики
- Развести расхождения в `names` между кодом и документами
- Дублировать полные политики в README или AGENTS.md
- Менять workflow в правилах, не сверившись с [AGENTS.md](../../AGENTS.md)
- Забыть `npm run sync:cli2-config` (обновить `.markdownlint-cli2.jsonc`) при добавлении/удалении custom lint-правила
- Обновить `.cursor/rules/*.mdc`, не обновив соответствующий `.claude/rules/*.md` (и наоборот)
