# Синхронизация правил Cursor ↔ Claude

> Claude-эквивалент [`.cursor/rules/rules-sync.mdc`](../../.cursor/rules/rules-sync.mdc). Применяется всегда.

Правила проекта существуют в двух параллельных наборах: [`.cursor/rules/*.mdc`](../../.cursor/rules/) (Cursor) и [`.claude/rules/*.md`](../rules/) (Claude Code, точка входа — [`CLAUDE.md`](../../CLAUDE.md)). Оба набора несут **один и тот же канон** политик; при правке любого правила в одном наборе — **в этом же шаге** зеркально обновить парный файл в другом наборе.

## Карта соответствия файлов

| Cursor (`.cursor/rules/`) | Claude (`.claude/rules/`) | Тема |
|----------------------------|----------------------------|------|
| `markdownlint-project.mdc` | `markdownlint-project.md` | Проект: структура, каталог правил, политики, конфиг lint |
| `platform-scripts.mdc` | `platform-scripts.md` | Кроссплатформенный запуск, bootstrap |
| `ts-style.mdc` | `ts-style.md` | Оформление `src/**/*.ts` |
| `ts-dev.mdc` | `ts-dev.md` | Проектирование TS-модулей |
| `js-style.mdc` | `js-style.md` | Оформление `test-rules.cjs` |
| `js-dev.mdc` | `js-dev.md` | Проектирование `test-rules.cjs` |
| `comments-style.mdc` | `comments-style.md` | Стиль комментариев в коде и выравнивание в docs |
| `docs-consistency.mdc` | `docs-consistency.md` | Синхронизация docs/код |
| `rules-sync.mdc` | `rules-sync.md` | Этот файл — синхронизация правил между редакторами |

Новое правило — добавить пару сразу в обоих каталогах и строку в этой таблице (в обеих версиях файла).

## Механические различия формата (не считаются расхождением)

Ниже — единственные допустимые отличия между парными файлами; всё остальное содержание (заголовки, порядок разделов, таблицы, примеры, формулировки политик) должно быть идентично по смыслу.

| Аспект | Claude `.md` | Cursor `.mdc` |
|--------|--------------|---------------|
| Frontmatter | Отсутствует; вместо него первая строка после `# Заголовок` — `> Claude-эквивалент [.mdc-файл](../../.cursor/rules/<file>.mdc). Применяется <когда>.` | YAML `description` / `globs` / `alwaysApply` |
| Расширение в кросс-ссылках | `[name.md](name.md)` | `[name.mdc](name.mdc)` |
| Глубина относительных путей к корню | `../../` (файл лежит в `.claude/rules/`) | `../` (файл лежит в `.cursor/rules/`; проверять фактическую глубину при правке, не копировать вслепую) |
| Упоминания `.md` в тексте (не в ссылке) | `.md` или нейтральное «правила» | `.mdc` |

## Обязательные семантические расхождения

`docs-consistency.md` / `docs-consistency.mdc` и `rules-sync.md` / `rules-sync.mdc` (этот файл) описывают саму двухредакторную схему — в них **сохраняются** взаимные ссылки друг на друга (Claude-файл ссылается на путь `.cursor/rules/...`, Cursor-файл — на `.claude/rules/...`) и текст о синхронизации не должен быть дословно зеркальным (естественно отличается по перспективе «из Claude» / «из Cursor»), но матрица изменений, чеклист и порядок работы должны покрывать один и тот же набор пунктов.

## Порядок работы при изменении правила

1. Внести правку в исходный файл (`.md` или `.mdc`);
2. Сразу перенести правку в парный файл (см. карту), адаптировав только механические различия из таблицы выше;
3. При добавлении/удалении правила — обновить таблицу карты соответствия в **обеих** версиях этого файла;
4. Прогнать `npm run lint:md -- README.md AGENTS.md CLAUDE.md` — оба каталога правил вне lint globs, но `AGENTS.md`/`README.md`/`CLAUDE.md` могут содержать ссылки на затронутые файлы;
5. Синхронизация с [docs-consistency.md](docs-consistency.md) / [docs-consistency.mdc](../../.cursor/rules/docs-consistency.mdc) — этот файл не отменяет её чеклист, а дополняет строкой «оба каталога правил обновлены».

## Антипаттерны

- Править `.md`, не перенеся эквивалентную правку в `.mdc` (и наоборот) в том же шаге, а не «потом»;
- Копировать frontmatter Cursor в Claude-файл или наоборот;
- Оставлять расхождение в карте соответствия при добавлении нового правила;
- Считать расхождением естественную разницу в оформлении frontmatter/расширений — не переносить эти технические детали как «правку по смыслу».

## Проверка перед завершением задачи, затрагивающей правила

- [ ] Каждый изменённый `.md` имеет актуальную пару `.mdc` (и наоборот) — сверить по карте выше;
- [ ] Единственные отличия между парой — из таблицы «Механические различия»;
- [ ] Карта соответствия актуальна в обеих версиях этого файла;
- [ ] `AGENTS.md` и `CLAUDE.md` ссылаются на новое/изменённое правило, если оно публичное для каталога.
